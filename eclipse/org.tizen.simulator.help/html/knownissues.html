Known Issues

There are a few known issues when running application in Web Simulator, and they are listed below and we are wokring on either a better solution or workarounds

  • Application will be loaded twice during refresh

  • Using instanceof to check if input argument is Array or Date wouldn't work.
    workaround: use the following snippet to type-check Date and Array. 
    isValidDate = function (d) {
    if (Object.prototype.toString.call(d) !== "[object Date]")
        return false;
    return !isNaN(d.getTime());
}

isValidArray = function (a) {
    return (Object.prototype.toString.call(a) === "[object Array]");
}

  • Several modules (e.g. filesystem, calendar, task..) are using database to simulate backend and this database has size limitation at 5M. Therefore, if total size of file/data is over 5M, Web Simulator would crash.
